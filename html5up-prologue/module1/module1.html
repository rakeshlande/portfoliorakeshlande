On day 2 we learned about CSS from basic

In CSS, selectors are used to target HTML elements for styling. Some common types of selectors include:

1. *Tag Selectors*: As mentioned earlier, these target HTML elements based on their tag names. For example: css p { /* styles */ }

2. *Class Selectors*: These target HTML elements based on their class attribute. css .class-name { /* styles */ }

3. *ID Selectors*: These target HTML elements based on their id attribute. css #id-name { /* styles */ }

4. *Attribute Selectors*: These target HTML elements based on their attributes. css [attribute=value] { /* styles */ }

5. *Pseudo-classes and Pseudo-elements*: These target elements in special states or positions. css selector:pseudo-class { /* styles */ } selector::pseudo-element { /* styles */ }

6. *Descendant Selectors*: These target elements that are descendants of another element. css ancestor descendant { /* styles */ }

7. *Child Selectors*: These target elements that are direct children of another element. css parent > child { /* styles */ }

These are some of the basic CSS selectors, but there are many more advanced selectors and techniques available for targeting specific elements in HTML documents.

javascript let x = 5;

3. *Operators*: You can perform arithmetic, assignment, comparison, and logical operations.

4. *Functions*: Functions are blocks of code that can be called and executed. Example:
javascript
function greet(name) {

}

5. *Conditional Statements*: You can use if, else if, and else statements for decision.

6. *Loops*: JavaScript supports for, while, and do-while loops for iteration.

7. *Arrays*: Arrays can hold multiple values of any data type. Example:
javascript
let fruits = ["Apple", "Banana", "Orange"];

8. *Objects*: Objects store key-value pairs and are used to represent complex data structures.

let person = {
name: "John",

2. *Create a New Repository*: Click on the "+" sign on the top right corner of your GitHub dashboard and select "New repository." Give your repository a name, like "my-website."

3. *Initialize with a README (Optional)*: You can choose to initialize your repository with a README file, or you can do it later.

4. *Upload Your Files*: Now, you need to upload your webpage files to your repository. You can do this by clicking the "Add file" button and selecting "Upload files." Then, drag and drop your webpage files into the upload area.

5. *Commit Changes*: After uploading your files, you need to commit your changes. Write a brief message explaining what you've added or changed, and then click on the "Commit changes" button.

6. *Set Up GitHub Pages*: Go to the settings tab of your repository, scroll down to the GitHub Pages section, and choose the branch you want to use (usually "main" or "master") and the folder from which GitHub Pages will serve your website.
